# “社保养老” VS “个人投资养老”计算器：项目交接

最后更新：2026-07-18

## 1. 项目现状

这是一个面向普通用户的中文养老方案计算器，用于比较：

1. 参加北京企业职工基本养老保险；
2. 把相当于个人养老保险缴费部分的资金投入个人投资账户。

当前可交付页面是一个完全独立的 HTML 文件：

- `outputs/两种养老方案_老人易懂版.html`

页面不依赖服务器、数据库、构建工具或外部 JavaScript 库。CSS、计算模型、交互逻辑和 SVG 图表全部内嵌在这一个文件里，适合直接部署到 GitHub Pages。

当前文件约 43 KB。退休模型测试共 17 项，最近一次运行全部通过。

## 2. 一个必须先处理的发布问题

当前目录原本是一个 Vinext/Next.js/Cloudflare 的通用脚手架，但养老计算器并没有集成进 `app/`。`app/page.tsx` 仍然只是默认的加载占位页。

更重要的是，当前 `.gitignore` 明确忽略了：

```text
/outputs/
```

因此，如果直接把现在的目录推送到 GitHub：

- 真正的养老计算器 HTML 默认不会进入 Git；
- GitHub Pages 也找不到站点入口；
- `npm test` 测试的是旧脚手架，而不是养老计算器。

### 推荐做法

为这个计算器建立一个干净、独立的静态网站仓库，只保留真正需要发布的文件：

```text
retirement-calculator/
├── index.html
├── README.md
├── HANDOFF.md
├── .nojekyll
└── tests/
    └── retirement-page.test.mjs
```

把当前页面复制并改名为 `index.html`。GitHub Pages 会把发布源目录中的 `index.html` 作为站点入口。GitHub 官方支持从指定分支的仓库根目录或 `/docs` 目录发布；每次向该分支推送后，站点会重新发布：

- [创建 GitHub Pages 站点](https://docs.github.com/en/pages/getting-started-with-github-pages/creating-a-github-pages-site)
- [配置 GitHub Pages 发布源](https://docs.github.com/en/pages/getting-started-with-github-pages/configuring-a-publishing-source-for-your-github-pages-site)

`.nojekyll` 是一个空文件，用来明确跳过 Jekyll 处理。当前页面不依赖 Jekyll。

不建议把当前整个通用脚手架和 `node_modules` 当作这个静态网站的正式仓库。虽然 `.gitignore` 已排除 `node_modules`，但其余 Next.js、Cloudflare、D1 和预览代码也与本项目无关，会增加维护成本。

## 3. 页面结构

页面分成五个主要区域：

### 顶部概览

- 标题：“社保养老” VS “个人投资养老”计算器；
- 两个核心退休月收入数字；
- 长期情景测算免责声明。

### 左侧参数区

用户可以调整：

- 开始工作年龄；
- 退休年龄；
- 当前社保缴费基数；
- 工资/缴费基数增长率；
- 长期通胀率；
- 社保个人账户记账利率；
- 个人投资年化回报；
- 个人账户养老金计发月数；
- 退休后养老金调整方式；
- 个人投资账户提款相对社保收入的倍数；
- 是否折算为今天购买力；
- 退休后是否继续投资。

参数变化时，全部数字、结论和图表会立即重算。

### 黑色总结区

这里不再重复比较两个月收入数字，而是动态显示：

- 个人投资账户预计在哪个年龄耗尽；或
- 到 95 岁仍有多少余额。

耗尽时间按月计算。“退休后继续投资”开关、投资回报率、提款倍数和养老金调整都会影响结果。模型只计算到 95 岁，不继续向后外推。

### 图表

页面使用原生 SVG 动态生成两张图：

1. 完整养老生命周期：工作期资产积累，以及退休后的月收入、累计社保领取和个人投资剩余余额；
2. 个人投资回报敏感性：不同长期回报率对应的 95 岁剩余资产。

### 模型口径与公式

底部的折叠区域解释：

- 基础养老金和个人账户养老金；
- 个人投资账户积累和提款；
- 北京 2025 年养老金调整规则模拟；
- 简单养老金年增长模式；
- 今天购买力折现；
- “回本”的局限。

## 4. 代码结构

HTML 中有两个主要脚本区。

### `model-core`

它把模型暴露为：

```js
globalThis.RetirementModel
```

主要成员：

- `defaults`：默认参数；
- `normalize()`：数值范围和布尔值标准化；
- `adjustPension()`：退休后养老金年度调整；
- `calculate()`：工作期积累、退休待遇、提款、耗尽年龄和 95 岁余额；
- `sensitivity()`：个人投资回报敏感性数据。

模型内部按月计算投资积累和退休提款，图表按年度取样展示。

### 页面交互脚本

第二个脚本负责：

- 读取表单参数；
- 调用 `RetirementModel.calculate()`；
- 更新核心数字和动态总结；
- 绘制生命周期图与敏感性图；
- 控制说明提示框；
- 恢复默认参数。

## 5. 当前默认参数

| 参数 | 默认值 |
|---|---:|
| 开始年龄 | 25 岁 |
| 退休年龄 | 60 岁 |
| 当前缴费基数 | 35,811 元/月 |
| 工资/基数年增长 | 2% |
| 长期通胀 | 2.37% |
| 个人账户记账利率 | 1% |
| 个人投资年化回报 | 8% |
| 计发月数 | 139 个月 |
| 养老金调整 | 简单年增长率 |
| 养老金年调整率 | 2% |
| 投资账户月提款 | 社保月收入的 1.3 倍 |
| 折算为当前购买力 | 关闭 |
| 退休后继续投资 | 开启 |

默认参数下，个人投资账户到 95 岁仍有余额。关闭退休后继续投资后，当前模型约在 73 岁 11 个月耗尽。这里采用固定年化回报，不代表真实市场路径。

养老计算器的正确测试命令是：


测试覆盖：

- 默认参数；
- 参数变化；
- 个人缴费指数；
- 社保与个人投资积累；
- 购买力折现；
- 养老金调整；
- 退休后继续投资开关；
- 账户耗尽年龄；
- SVG 图表所需结构；
- 离线、响应式和基础可访问性；
- 关键文案和风险提醒。

当前根目录的：

```bash

```

仍然测试旧的 Vinext 加载占位页，不代表养老计算器已经通过测试。迁移到干净仓库后，应删除旧脚手架测试，或把 `package.json` 的 `test` 命令改为上述养老模型测试。

## 8. 已知模型限制

发布时不要把结果描述为预测或保证。当前模型具有以下限制：

- 所有长期增长率都是固定情景值；
- 投资回报没有年度波动，也没有模拟熊市；
- “退休后继续投资”按照稳定月复利计算；
- 北京 2025 年养老金调整规则只作为未来情景模板；
- 简单养老金调整模式使用固定百分比；
- 个人账户记账利率 1% 是保守假设，不是官方未来承诺；
- 缴费上限除以 3 被用来近似当前社平工资；
- 模型没有完整处理未来退休政策、最低缴费基数和地区差异；
- 没有计算税费、基金管理费、汇率或交易限制；
- “今天购买力”按用户输入的固定通胀率逐期折现；
- 计算截至 95 岁，95 岁之后不再外推；
- 社保“回本”只是现金流比较，不是严格投资收益率。

## 9. GitHub Pages 发布建议

最简单的发布方式：

1. 创建公开 GitHub 仓库；
2. 把静态站点文件放在 `main` 分支根目录；
3. 确保入口名为 `index.html`；
4. 在 `Settings → Pages` 选择 `Deploy from a branch`；
5. 选择 `main` 和 `/(root)`；
6. 保存并等待 GitHub 给出公开地址。

GitHub Pages 支持 HTTPS；如果以后绑定自定义域名，应启用 HTTPS，并优先先验证域名，避免域名接管风险：

- [为 GitHub Pages 启用 HTTPS](https://docs.github.com/en/pages/getting-started-with-github-pages/securing-your-github-pages-site-with-https)
- [管理自定义域名](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site/managing-a-custom-domain-for-your-github-pages-site)
- [验证自定义域名](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site/verifying-your-custom-domain-for-github-pages)

## 10. 公开发布前的安全检查

最近一次针对独立 HTML 的检查没有发现 API Key、密码、用户本地绝对路径或 `file://` 链接。页面不收集表单数据，也没有网络请求；只有用户主动点击时才会打开外部资料链接。

但推送前仍应执行：

```bash
git status
git diff --cached
```

确认没有提交：

- `.env` 文件；
- API Key、访问令牌或密码；
- `node_modules`；
- 本地日志、缓存或构建产物；
- 与本项目无关的旧脚手架和数据库代码。
