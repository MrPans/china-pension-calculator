# SEO and AEO Optimization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the pension calculator consistently discoverable at its shengpan.net canonical URL and provide accurate, visible, machine-readable answers for search engines and AI agents.

**Architecture:** Keep the application dependency-free and static. Extend `index.html` with metadata, JSON-LD, and visible explanatory content; add three root-level discovery files; extend the existing Node test file to treat visible FAQ text and JSON-LD as one contract.

**Tech Stack:** HTML5, CSS, JSON-LD/Schema.org, XML sitemap, plain text Markdown, Node.js built-in test runner.

## Global Constraints

- The canonical URL is exactly `https://www.shengpan.net/china-pension-calculator/`.
- The page remains a dependency-free static site suitable for GitHub Pages.
- Structured data must describe visible page content and must not claim ratings, professional credentials, or guaranteed outcomes.
- FAQ rich results, search ranking, indexing, and AI citation are not promised.
- No social sharing image, `hreflang`, keyword meta tag, or IndexNow integration is added in this scope.
- Existing calculator behavior, model formulas, and hero insight remain unchanged.
