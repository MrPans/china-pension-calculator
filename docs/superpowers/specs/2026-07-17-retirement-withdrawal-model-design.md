# 退休提款组合图与购买力开关设计

## 目标

把生命周期图的退休阶段升级为按月模拟、按年度取样的柱状图与折线图组合，使用户能同时观察月现金流、社保累计领取和标普账户剩余资产。

## 新参数

- `withdrawalMultiple`：退休后标普账户目标月提款为社保月收入的倍数，默认1.3倍。
- `purchasingPower`：默认打开。打开时，从今天到每个年龄的所有图表金额按累计通胀折现；关闭时显示名义金额。
- `continueInvesting`：默认关闭。打开时退休后标普余额继续按同一标普年化回报逐月增长，关闭时退休后收益率为0。
- 原 `payoutMonths` 保留，但明确只用于社保个人账户养老金计发。

## 月度模型

退休模拟延伸至95岁。每月先按开关计算投资收益，再提款；实际提款不超过账户余额。社保月收入保持固定名义金额。购买力开关打开时，每个月的社保收入、实际标普提款和标普余额都用从今天至该月的累计通胀折现；累计社保领取额累加每月折现后的社保收入。

## 图表

生命周期上层继续显示工作期资产积累，并受购买力开关影响。下层使用双轴组合图：蓝色柱为月均社保收入，橙色柱为月均实际标普提款；蓝色折线为累计社保领取额，橙色折线为标普剩余资产。柱使用左轴，折线使用右轴，按年度取样。

敏感性图改为不同长期标普回报率下95岁标普账户剩余资产，并受购买力与退休后继续投资开关影响。

## 验证

- 自动验证三个新参数、年度退休序列和提款边界。
- 验证继续投资时默认情景的95岁余额高于停止投资情景。
- 验证较低提款倍数留下更多余额，购买力模式显示金额低于名义模式。
- 浏览器验证三个控件都能更新指标和图形，手机无溢出且无运行错误。
