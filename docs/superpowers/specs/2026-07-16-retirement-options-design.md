# 两种养老方案单页网页设计

## 目标与受众

制作一份中文、离线可打开的单页网页，让没有金融知识、视力可能较弱的老人理解两种养老安排的核心差异。网页以少量关键数字、生活化比喻和清晰风险提示为主，公式与详细口径默认折叠。

## 内容结构

1. 首屏：一句话结论、共同假设、方案 A 与方案 B 的今天购买力。
2. 两个养老钱包：分别解释社保的社会大水池与私人投资箱。
3. 钱从哪里来：个人 8%、单位 16%、个人投资 8% 的去向。
4. 退休后拿多少：名义金额与今天购买力分开显示。
5. 能拿多久：社保通常终身；投资基准方案约至 71 岁 7 个月。
6. 直接比较：用短句卡片和表格对照投入、收入、遗产、波动和长寿保障。
7. 回本与收益情景：展示社保三种现金流比较口径，以及标普 6%、8%、10%、11.5% 情景。
8. 风险与结论：列出政策风险、市场风险、汇率风险、纪律要求与长期假设声明。
9. 可折叠“怎么算的”：收纳基础养老金、个人账户养老金、通胀折现与基金提款公式。

## 视觉与交互

- 暖米色背景、深蓝代表社保、橙红代表投资；正文保持深色高对比。
- 桌面双栏、手机单栏；正文不小于 18px，主要数字使用 32–56px。
- 使用 CSS 图形和文字图标，不依赖外部图片、字体或脚本。
- 原生 `details/summary` 提供键盘可用的折叠区域。
- 尊重减少动态效果设置，并提供打印样式。

## 数据口径

所有展示数字与用户给定口径一致：35 年、北京代表案例、缴费基数上限 35,811 元、增长与通胀均为 5.97%、社保月养老金 106,432 元、今天购买力 13,985 元；投资基准年化 10%、月提款 139,300 元、今天购买力 18,301 元、领取 139 个月。

## 验收

- 生成一个位于 `outputs/` 的独立 HTML 文件，断网亦可打开。
- 自动检查标题、关键数字、风险提示、折叠区、移动端样式和无外部资源依赖。
- 用本地浏览器渲染检查桌面与手机布局，验证折叠区可操作。
