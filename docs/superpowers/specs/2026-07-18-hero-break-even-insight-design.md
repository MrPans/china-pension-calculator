# 顶部临界点洞察改版设计

## 目标

消除顶部蓝色区域与下方 KPI 对“社保退休月收入”和“个人投资月提款”的重复展示，让首屏承担更有价值的决策解释：在代表寿命下，个人投资需要达到多高的长期年化回报，家庭累计价值才能追平社保。

## 顶部布局

顶部继续使用现有深蓝背景、光晕、品牌栏和两列布局。

### 左侧

- 保留标题：`“社保养老” VS “个人投资养老”计算器`；
- 标题最大字号从当前 62px 适度缩小，使右侧获得更多空间，但保持其为首屏最高层级；
- 副标题改为：`调整参数使其符合您目前的情况，再对比哪种养老方案更适合您。`

### 右侧

用一个半透明洞察卡替换当前两个月收入卡片：

- 标签：`家庭累计价值 · 临界点洞察`；
- 代表寿命徽标：`代表寿命 85 岁`；
- 主结论：`个人投资需达到约 X% 年化回报，才能追平社保`；
- 判断说明：低于 X% 时社保侧家庭累计价值更高，高于 X% 时个人投资侧更高；
- 参考点：70、75、80、85、95 岁对应的临界回报率；
- 风险提示：`寿命越长，个人投资需要达到的临界回报率越高。固定收益情景，不构成投资建议。`

85 岁是主视觉代表点，不增加新的“预期寿命”输入框。参考点用于展示寿命维度，不作为交互控件。

## 动态计算口径

所有临界回报率随现有参数变化而重算，不硬编码 README 中的默认数值。

家庭累计价值沿用 README 已定义的口径：

- 社保侧 = 截至目标年龄累计领取的养老金 + 当时尚可继承的社保个人账户余额；
- 个人投资侧 = 截至目标年龄累计提款 + 当时个人投资账户剩余余额；
- 单位缴纳的统筹基金不作为个人遗产；
- 不计税务、丧葬补助、抚恤金、基金费率和其他社保待遇。

模型对候选个人投资回报率进行 0%–20% 区间求根，寻找两侧家庭累计价值相等的年化回报率。若临界点在区间外，界面显示“低于 0%”或“高于 20%”，不显示误导性的区间内数字。

## 模型接口

在 `RetirementModel` 中增加两个可测试接口：

- `familyValueAtAge(params, deathAge)`：返回指定寿命下社保侧与个人投资侧家庭累计价值，以及个人投资耗尽时间；
- `breakEvenReturnAtAge(params, deathAge)`：返回 0%–20% 内的临界回报率或区间外状态。

页面 `update()` 在现有参数改变时计算 70、75、80、85、95 岁五个点，并更新洞察卡。计算只在输入事件发生时运行，不添加后台任务或网络请求。

## 响应式与可访问性

- 桌面端保持左右两列，洞察卡宽度不小于 320px；
- 960px 以下沿用现有单列布局，洞察卡排在标题下方；
- 参考点允许均匀压缩或换行，320px 宽度下不得截断；
- 主结论使用现有文本对比度，动态区域使用 `aria-live="polite"`；
- 不依赖颜色表达方向，文字明确写出“社保侧”和“个人投资侧”。

## 测试与验收

- 默认参数下，85 岁临界回报率约为 7.01%，70/75/80/95 岁分别约为 4.33%/5.61%/6.47%/7.61%；
- 修改工资增长、养老金调整、记账利率、提款倍数或退休后继续投资开关后，临界点同步更新；
- 顶部不再显示与下方 KPI 重复的两个月收入数字；
- 左侧标题文字不变，副标题与用户确认文案完全一致；
- 原有计算、图表、ICP 链接和输入框布局测试继续通过；
- 桌面与移动宽度均无文字截断或布局溢出。
