# 当前缴费基数输入框数字截断修复设计

## 问题

“当前缴费基数”输入框默认显示五位数 `35811`，但所在字段误用了 `.wide-unit`。该类为“个月”等较宽单位预留 `40px` 右内边距；当前字段的单位只有“元”，多余留白与数字输入框原生步进按钮共同压缩了数字显示区域。

## 方案

仅从包含 `#baseCap` 的字段移除 `.wide-unit`，使它恢复普通输入框的 `29px` 右内边距。保留“社保个人账户计发月数”字段的 `.wide-unit`，不调整全局网格、字体或其他字段。

## 验收

- `#baseCap` 所在字段不再包含 `.wide-unit`；
- `#payoutMonths` 所在字段继续包含 `.wide-unit`；
- 现有 ICP 页脚测试及 HTML 内嵌脚本语法检查继续通过；
- 改动提交并推送至 `main`。
