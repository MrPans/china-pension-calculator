# 年轻用户养老方案互动测算器设计

## 设计选择

采用“金融数据工作台”方向，而不是故事化科普页：深色首屏、紧凑排版、荧光蓝与珊瑚橙区分两种方案，数字和图表成为视觉主体。用户已明确要求直接开始，因此以此方案作为获批设计。

另外两个可选方向不采用：长文章式数据新闻更适合阅读但交互弱；全屏专业终端信息密度过高，不适合普通年轻用户。

## 交互结构

1. 顶部展示社保与标普方案的“今天购买力”、领取期限和当前领先方案。
2. 参数区提供开始年龄、退休年龄、当前缴费上限、缴费基数增长率、通胀率、个人账户利率、标普回报率和领取月数；输入即重算，带恢复默认值按钮。
3. 图表一：工作期间资产积累曲线，对比社保个人账户与标普投资账户。
4. 图表二：退休后月收入随年龄变化，突出社保延续与私人账户耗尽。
5. 图表三：标普回报率敏感性曲线，并标注与社保购买力的交点附近。
6. 下方保留资金去向、风险、回本和可折叠公式，但删除老年化措辞与大字故事风格。

## 计算模型

- 每年缴费基数按增长率增长，第一年使用“当前基数×(1+增长率)”。
- 每月个人投入为当年缴费基数×个人缴费率；单位缴费率固定为个人的两倍（16%）。
- 社保个人账户与标普账户都按月复利、月末投入。
- 基础养老金 = 退休时社平工资 × (1+缴费指数3)/2 × 缴费年限 × 1%。
- 个人账户养老金 = 个人账户余额÷领取月数。
- 私人账户月提款 = 标普账户余额÷领取月数，退休后余额不再计息。
- 今天购买力使用通胀率折现整个缴费年限。

## 验证

- 自动验证默认参数大致复现原始测算：个人累计投入约403万元、社保约10.64万元/月、标普10%约13.93万元/月。
- 自动验证参数变化会改变结果，页面包含8个输入和3张动态图表。
- 浏览器验证输入联动、重置、图表更新、手机无横向溢出、无控制台错误。
