# 北京退休养老金年度调整模型设计

## 目标

在现有养老方案模拟器中加入退休后养老金年度增长，使退休阶段不再假设社保养老金名义金额永久不变。模型同时提供两种预测方式，并明确未来政策不可准确预知。

## 官方规则依据

北京市职工基本养老金的年度调整不是统一乘以某个比例，而是采用“定额调整、挂钩调整、适当倾斜”的组合。

- 2025年北京：定额18元；缴费满10年以上者每年缴费年限增加1.8元；养老金高于7,118元者增加3元、低于或等于该值者增加18元；按上年12月养老金增加0.15%；65岁起按年龄增加10至50元；65岁以上且缴费满30年再增加5元。
- 2024年北京：定额30元；缴费年限每年增加2.4元；养老金分档增加15元或30元；按原养老金增加0.2%；65岁起另有20至60元高龄倾斜及缴费满30年的5元倾斜。
- 国家公布的总体调整水平是平均口径，不等于个人养老金直接按该比例增长。2025年全国总体调整水平为2%。

页面底部“模型口径与公式”将写明上述区别，并列出北京市人社局、北京市政府和中国政府网的官方链接。

- 北京2025年规则：https://rsj.beijing.gov.cn/xxgk/2024zcjd/202508/t20250801_4164801.html
- 北京2024年规则：https://www.beijing.gov.cn/zhengce/gfxwj/202406/t20240629_3733220.html
- 国家2025年调整通知：https://www.gov.cn/zhengce/zhengceku/202507/content_7031449.htm

## 新增配置

### 养老金调整方式

下拉选择：

1. `北京2025规则模拟`（默认）
2. `简单年增长率`

选择简单年增长率时显示“退休后养老金年调整率”，默认2%，允许0%至10%。

### 北京规则模拟

年龄60岁的样本使用退休首月养老金，不立即调整。从下一个完整退休年度开始，每年年初调整一次：

```text
当年月增额
= 定额调整
+ 缴费年限挂钩
+ 养老金分档调整
+ 上年末养老金 × 0.15%
+ 高龄倾斜
+ 高龄且长缴费倾斜
```

本案例缴费年限为工作年数。高龄倾斜从65岁开始：65至69岁10元，70至74岁30元，75至79岁40元，80岁及以上50元；65岁以上且缴费满30年再增加5元。

由于未来几十年的固定金额和分档线未知，模型将2025年的人民币固定金额与7,118元分档线按“工资/基数年增长率”逐年放大；第一个调整年度的放大年数等于“工作年数+1”，此后每年递增一年。0.15%的挂钩比例保持不变。这是结构化情景外推，不是政策预测。

### 简单年增长率

每个完整退休年度按用户输入的增长率复利：

```text
当年养老金 = 上年养老金 ×（1 + 退休后养老金年调整率）
```

## 退休期数据流

1. 先计算退休首月名义社保养老金。
2. 每个退休年度开始时，按所选方式更新名义社保养老金。
3. 当年标普目标月提款额始终等于“当年社保月养老金 × 用户提款倍数”。
4. 标普账户仍按月计算收益和提款，实际提款不超过账户余额。
5. 打开“折算为当前购买力”后，每个月的社保收入、标普提款、累计领取和账户余额均按该月距离今天的时间折现。
6. 图表继续按年度取样，内部计算保持逐月。

## 页面展示

- 左侧增加养老金调整方式；简单模式下显示调整率输入框。
- 退休收入蓝色柱随年度养老金调整变化。
- 橙色提款柱随当年社保养老金及提款倍数变化。
- 图表摘要说明当前采用的调整方式。
- 底部新增“退休后养老金为什么会增长”折叠区域，展示两种公式、北京2025年具体组成、高龄倾斜和未来外推限制。

## 测试

- 北京规则模式下，61岁养老金高于60岁养老金。
- 简单2%模式下，下一年度名义养老金约为上一年度的1.02倍。
- 当养老金调整率与通胀率相同时，简单模式的退休月收入今天购买力基本保持稳定。
- 北京规则的65、70、75和80岁年龄档出现对应倾斜。
- 标普目标提款随当年社保养老金同步变化，但不超过账户余额。
- 购买力开关、退休后继续投资开关及既有默认结果继续正常工作。
- 页面底部存在官方规则说明和来源链接。
